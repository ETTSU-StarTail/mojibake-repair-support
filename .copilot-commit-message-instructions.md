# コミットメッセージ規約（Conventional Commits + Gitmoji）

- メッセージは必ず日本語で記述
- Conventional Commits形式＋Gitmoji（絵文字）に準拠
- フォーマットは必ず次の通りにしてください：
  ```
  <type>(<scope>): <絵文字> <短い説明（50字以内）>

  - <変更の背景や理由、使用している技術や意図、将来の理解に役立つ情報を補足（短文で箇条書き）>

  <フッター（必要に応じて）>
  ```
- type/scope/絵文字は下表から選ぶ（詳細はリファレンス参照）
- 破壊的変更は `!` またはフッターに `BREAKING CHANGE:` で明示
- 問題やタスクを修正する場合は、フッターに `Fixes #<issue-number>` または `Closes #<issue-number>` を記載してください。

## 破壊的変更（BREAKING CHANGE）

後方互換性を壊す変更を含む場合は、以下のいずれかの方法で明示する。

**方法 1: `!` 記法**（概要のみで伝わる場合）
```
<type>(<scope>)!: <絵文字> <概要メッセージ>
```

**方法 2: フッターに `BREAKING CHANGE:` を記述**（詳細な説明が必要な場合）
```
<type>(<scope>): <絵文字> <概要メッセージ>

<詳細メッセージ>

BREAKING CHANGE: <後方互換性が壊れる内容と移行方法>
```

## type 一覧

type は必ず下表を参照して、変更内容と合致する type を選定してください。
※日本語訳は参考情報です。

| type | 説明 | 日本語訳 |
|------|--------------|-----------|
| build | Changes that affect the build system or external dependencies (example scopes: gulp, broccoli, npm) | ビルドシステムや外部依存関係に影響する変更（例：gulp, broccoli, npm） |
| ci | Changes to our CI configuration files and scripts (example scopes: Travis, Circle, BrowserStack, SauceLabs) | CI の設定ファイルやスクリプトに対する変更（例：Travis, CircleCI, BrowserStack, SauceLabs） |
| docs | Documentation only changes | ドキュメントのみの変更 |
| feat | A new feature | 新しい機能 |
| fix | A bug fix | バグ修正 |
| perf | A code change that improves performance | パフォーマンスを改善するコード変更 |
| refactor | A code change that neither fixes a bug nor adds a feature | バグ修正でも機能追加でもないコード変更 |
| style | Changes that do not affect the meaning of the code (white-space, formatting, missing semi-colons, etc) | コードの意味に影響しない変更（空白、フォーマット、セミコロン不足など） |
| test | Adding missing tests or correcting existing tests | テストの追加または既存テストの修正 |
